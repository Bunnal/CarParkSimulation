┌─────────────────────────────────────────────────────────────────────────────┐
│                    KEY DESIGN PATTERNS USED                                  │
└─────────────────────────────────────────────────────────────────────────────┘

1. PRODUCER-CONSUMER PATTERN
   └─ Multiple threads sharing bounded buffer
   
2. MONITOR PATTERN (via ReentrantLock + Conditions)
   └─ Encapsulate synchronization within CarPark
   
3. MODEL-VIEW-CONTROLLER (MVC)
   └─ Model: Car, CarPark
   └─ View: DashboardUI
   └─ Controller: CarOwner, SecurityGuard, SimulationManager
   
4. OBSERVABLE PATTERN (implicit)
   └─ AnimationTimer polls for updates
   
5. FACTORY PATTERN (implicit)
   └─ SimulationManager creates thread instances


┌─────────────────────────────────────────────────────────────────────────────┐
│                      CONCURRENCY GUARANTEES                                  │
└─────────────────────────────────────────────────────────────────────────────┘

✓ MUTUAL EXCLUSION
  └─ ReentrantLock ensures only one thread modifies buffer at a time

✓ NO RACE CONDITIONS
  └─ All shared state protected by locks

✓ NO DEADLOCK
  └─ Single lock, consistent acquire/release order

✓ NO STARVATION
  └─ Fair lock provides FIFO ordering

✓ NO BUSY-WAITING
  └─ Condition variables block threads efficiently

✓ MEMORY CONSISTENCY
  └─ Lock provides happens-before relationship

✓ GRACEFUL SHUTDOWN
  └─ Interrupt handling + volatile flags
